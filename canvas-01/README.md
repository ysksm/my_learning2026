# Canvas ダッシュボード開発 学習コース

HTML5 Canvas でダッシュボード(ウィジェットの配置・移動・リサイズができる画面)を
作れるようになるための、ステップバイステップ教材です。

## 進め方

各ステップのフォルダには 3 つのファイルがあります。

| ファイル | 役割 |
|---|---|
| `README.md` | そのステップの解説。使うメソッドの意味と仕組みを深く説明 |
| `demo.html` | 完成して動くサンプル。ブラウザで開いて動きを確認し、コードを読む |
| `exercise.html` | `TODO` コメント入りの穴埋め演習。自分で書いて完成させる |

推奨の流れ:

1. `README.md` を読む
2. `demo.html` をブラウザで開き、動きとコードを対応させて理解する
3. `exercise.html` を自分で完成させる(答えは README 末尾の「解答例」)
4. 余裕があれば demo を改造して遊ぶ(理解が一番深まるのはここ)

HTML ファイルはすべて自己完結(外部ライブラリなし)なので、
ダブルクリックでブラウザで開くだけで動きます。
ローカルサーバを使う場合は `python3 -m http.server` などでも OK です。

## カリキュラム

| Step | テーマ | 学ぶこと |

| [step01](step01-canvas-size/) | キャンバスのサイズ | width/height 属性と CSS サイズの違い、ぼやけ問題、devicePixelRatio、リサイズ対応 |
| [step02](step02-drawing-basics/) | 描画の基本 | 矩形・パス・テキスト、fillStyle/strokeStyle、描画状態という考え方 |

| [step04](step04-world-coords/) | 仮想座標とビューポート | ワールド座標⇔スクリーン座標の変換、パン、マウス位置中心のズーム |
| [step05](step05-hittest/) | マウスとヒットテスト | マウス座標の正しい取得、矩形ヒットテスト、isPointInPath、前面優先 |
| [step06](step06-drag/) | ドラッグ移動 | mousedown/move/up の設計、つかんだ位置のオフセット、再描画ループ |

## このコースの設計方針

- **1ステップ1テーマ**。複数の概念を同時に導入しない
- ダッシュボードに **本当に必要な技術だけ** を扱う(画像処理やゲーム向けの話はしない)
- 各ステップは前のステップの知識だけで理解できるように積み上げる
- 「なぜそう書くのか」(仕組み)を必ず説明する。暗記させない

## 最終的に作れるもの (step09)

- グリッド付きの無限キャンバス(パン・ズーム可能)
- 複数のウィジェット(時計、棒グラフ、カウンタ)を配置
- ウィジェットをクリックで選択、ドラッグで移動、ハンドルでリサイズ
- 半透明のパネルやホバーエフェクト
